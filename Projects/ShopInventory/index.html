Project Overview

This project implements a robust backend for an RPG economy. It handles the complexities of a Player Inventory and a Merchant Shop, ensuring that all transactions adhere to strict gameplay rules such as weight capacity, stack limits, and currency availability.

The primary goal was to decouple the Business Logic (Data/Models) from the Presentation Layer (Unity UI), making the system testable and easily expandable.

Architecture: Core, Logic, & Data

The codebase is organized into distinct layers to enforce separation of concerns:

  • Core: Handles initialization and Dependency Injection via a CentralRegistry.
  • Logic: Pure C# classes (PlayerInventory, ShopInventory) that manage state and fire events.
  • Data: ScriptableObjects defining Item properties (Weight, Rarity, Price).
  • UI: A reactive view layer that observes the Logic layer for changes.
